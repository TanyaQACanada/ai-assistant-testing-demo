Testing an AI assistant is not about one perfect answer.

It is about four things we can actually check:

1. **Grounding** — Does it answer only from allowed context and avoid inventing facts?
2. **Fallback** — When it does not know, does it say so clearly instead of guessing?
3. **Safety** — Does it refuse harmful or policy-breaking requests (e.g. prompt injection)?
4. **Consistency** — Do similar questions get compatible answers?

I put together a small public project to show how to test these in code: evaluation scenarios, assertion helpers, and a mock assistant so everything runs without a real LLM. Link in the comments if you want to try it.

If you work in QA or product and you are starting to test AI features, I hope this gives you a clear place to start. The same ideas work for chatbots, copilots, and any assistant that has to be useful, safe, and honest.
